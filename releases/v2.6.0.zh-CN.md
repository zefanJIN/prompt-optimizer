# Prompt Optimizer v2.6.0

## 概括
- 新增了 MiniMax provider 支持，也让高级模型配置的管理更顺手。
- 临时变量编辑、模型管理和 XML 结果查看体验都有明显改进。
- 多个工作区之间的模型同步和反馈链路也更稳定了，日常迭代更顺畅。

## 亮点
- MiniMax 成为一等公民 provider，更方便在同一工作区里测试和运行更多模型组合。
- 模型管理器和临时变量工具更适合高频提示词迭代场景，操作成本更低。
- XML 输出现在会自动渲染为可折叠树视图，查看结构化结果更直观。

## 产品更新
### Web
- 优化了模型管理器的布局和卡片设计，让文本、图像、函数模型的呈现更统一、更容易浏览。
- 改进了临时变量面板，补齐了更顺手的编辑体验、变量重命名能力，以及更充分利用弹窗空间的全屏编辑器。
- 新增 XML 自动渲染与可折叠树视图，不用离开应用也能更轻松地检查结构化输出。
- 模型管理器更新后，模型列表可以更稳定地同步到各个工作区，同时也收紧了反馈数据的类型处理。

### Core/Infra
- 新增 MiniMax provider 支持，包括兼容 OpenAI 风格接口的接入方式和初始默认模型配置。
- 去重了共享的 `TextModelConfig` 类型来源，后续维护模型配置逻辑更清晰。
- 优化了模型启用状态的保留逻辑，并统一了版本标签格式。

## 修复
- 修复了模型管理器修改后其他工作区模型列表不同步的问题。
- 修复了全屏变量编辑器里的滚动跳动问题。
- 修复了模型管理器中的若干交互细节，包括启用状态保留和版本标签显示不清晰的问题。

## 破坏性变更 / 升级说明
- 无。现有工作区和已保存数据默认不需要迁移。

## 开发者说明
- 这一版是后续 `v2.6.1`、`v2.6.2` 模型管理和评估链路继续演进的重要基础版本。
- 如果你要追溯 MiniMax provider 支持和 XML 树形查看器最早在哪个版本落地，可以以 `v2.6.0` 为基线。
