# Prompt Optimizer v2.8.0

## 概括
- 文生图工作流进一步增强，补齐了多模态评估、结果评估和原始提示词分析能力。
- 参考图相关流程顺了很多，从对话框设计、提示词迁移到更安全的预览与下载都做了打磨。
- 模型接入和打包底座也更稳了，新增了基于 Cloudflare 的 provider，同时让桌面端发布链路更可靠。

## 亮点
- 文生图工作流进一步增强，补齐了多模态评估、结果评估和原始提示词分析能力。
- 参考图相关流程顺了很多，从对话框设计、提示词迁移到更安全的预览与下载都做了打磨。
- 模型接入和打包底座也更稳了，新增了基于 Cloudflare 的 provider，同时让桌面端发布链路更可靠。

## 产品更新
### Desktop
- 桌面端打包工具链升级到 `electron-builder 26.8.2`。
- 移除了仓库级 Electron mirror 默认配置，避免 CI 发布构建被错误环境变量干扰。

### Web
- 重做了文生图参考图对话框，让风格迁移和参考图复制这类操作更直接。
- 把新的文生图评估工作流接入到了 UI，包括围绕图片生成的分析和评估入口。
- 改进了预览下载体验、临时变量状态持久化，以及页头、浮层、路由守卫等一批交互细节。

### Core/Infra
- 新增了基于 Cloudflare 的文本与图像 Provider，并补齐多模态文生图评估能力。
- 重建了参考提示词处理链路，更好地支撑提示词组合、迁移、对比评估和结果评估。
- 规范了旧模板标识，并把证据内容包装成结构化 JSON 块，让评估行为更稳定一致。

## 修复
- 修复了预览下载按钮点击区域问题，并统一了 URL 图片结果的处理。
- 加固了安全图片预览下载流程，并稳定了基于 Cloudflare 的参考图链路。
- 回填了内置模型缺失的 API Key 配置，同时让评估夹具与新的图像工作流保持一致。

## 破坏性变更 / 升级说明
- 无。现有保存数据与当前工作区默认不需要额外迁移步骤。

## 开发者说明
- 这一版对图像提示词链路做了较大幅度重整。如果你维护自定义模板或评估提示词，需要关注更结构化的证据块，以及更明确的阶段感知评估路径。
- 从 `v2.7.0` 到 `v2.8.0` 的提交范围里也包含仓库清理、依赖对齐和文档整理，但真正面向用户的主轴还是图像评估能力的提升。
