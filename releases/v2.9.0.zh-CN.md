# Prompt Optimizer v2.9.0

## 概括
- 首个端到端多图生图工作流已经落地，覆盖有序图片输入、独立工作区、多模态提示词优化以及 provider 侧适配。
- 本地存储与收藏安全性进一步加固，新增启动自检、收藏图片引用清理、容量防线，以及更清晰的数据管理存储概览。
- 图像链路中一些过时的本地前置拦截被移除，串并行执行、会话恢复、E2E 回放稳定性和双语 release 发布链路也一起补强。

## 亮点
- 多图生图不再只是孤立的 UI 能力，而是贯通了优化、生成、收藏、桌面桥接和会话恢复的完整链路。
- 存储安全防线明显加强：启动时可以修复异常状态，收藏图片孤儿资源会被主动回收，超大或畸形数据也更难继续静默膨胀。
- 数据管理面板现在能更直观地展示不同存储区域和大致占用，有助于在启动失败或持久化异常之前提前发现风险。

## 产品更新
### Desktop
- 在恢复会话前增加启动期存储安全检查，尽量避免损坏的设置、收藏或 session 数据直接拖垮启动流程。
- 扩展了桌面端图像桥接能力，让多图资源在持久化和恢复链路里更稳定。

### Web
- 新增重设计的多图工作区，补齐有序输入管理、路由与会话守卫、token 用量反馈、历史恢复和 Prompt Garden 集成。
- 优化了数据管理中的存储概览，让用户能更直接地看到不同存储区域及其大致体量。
- 恢复了自动化场景之外的并行变体执行能力。

### Core/Infra
- 新增多图请求类型、模板和 provider 路由，并补齐 OpenRouter 请求体修复与 Seedream 多图生成支持。
- 多图会话输入改为以资源引用方式持久化，而不是继续写入内联大 payload，恢复行为也更可控。
- 新增收藏存储防线、收藏图片 GC，以及基于仓库文件的双语 release 说明链路。

## 修复
- 修复了 OpenRouter 多图请求没有把有序输入图片真正序列化进 provider payload 的问题。
- 修复了过时本地 capability 判断会在请求发出前错误拦截多图生图的情况。
- 修复了多图会话恢复时图片资产缺失却静默降级为空输入的问题，改为明确失败，避免坏数据继续进入链路。
- 加固了收藏图片配额、导入覆盖和启动恢复链路，降低孤儿数据与静默膨胀的风险。
- 稳定了 gated E2E 回放流程，并刷新了相关图像夹具，回归覆盖更可靠。

## 破坏性变更 / 升级说明
- 对合法数据和正常工作流没有破坏性变更。
- 启动期修复会比以前更积极：异常 session、非法收藏条目或未被引用的收藏图片资源，可能会在启动时被自动清理，以保证应用可启动且存储体量受控。
