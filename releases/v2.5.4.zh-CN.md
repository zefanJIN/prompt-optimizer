# Prompt Optimizer v2.5.4

## 概括
- 评估流程开始支持“基于反馈继续分析”，用户可以直接带着明确意见去驱动评估。
- 评估模板更加重视用户反馈，同时结果解析也更稳健了。
- 分析交互流被顺了一遍，provider 环境变量映射也补得更完整。

## 亮点
- 评估不再只是看完分数就结束，用户现在可以把反馈直接带入评估流程，形成更连续的迭代闭环。
- 评估底层把用户反馈视为更强的信号，让后续分析结果更贴近真实诉求。
- 结果解析和 provider 配置都更稳了，减少了高频评估场景里的额外摩擦。

## 产品更新
### Web
- 在主要工作区里新增了评估反馈编辑器，以及基于反馈继续分析的交互流。
- 把 focus input 更自然地并入 analyze 动作，让用户能在合适的时机补充评估指导信息。
- 执行 apply 操作时，评估浮层会保持打开，连续迭代时不再那么打断思路。

### Core/Infra
- 为评估模板全面补上了用户反馈支持，并让反馈在评估时拥有更高优先级。
- 增强了评估结果解析的鲁棒性，让结构化输出的解释更稳定一致。
- 补齐了缺失的 provider 环境变量映射，并对齐了桌面日志、MCP 选择和文档中的 provider 处理逻辑。

## 修复
- 修复了若干会削弱后续分析质量的评估结果解析边界情况。
- 修复了部分依赖环境变量驱动的模型配置缺少 provider 映射的问题。
- 修复了评估浮层在 apply 操作时过早打断迭代查看的问题。

## 破坏性变更 / 升级说明
- 无。现有评估流程可以继续使用，只是现在对反馈的支持更完整了。

## 开发者说明
- `v2.5.4` 可以看作评估体系从“单向打分展示”转向“反馈驱动闭环”的起点版本。
- 如果你维护自定义评估提示词或结果消费逻辑，这一版是更强反馈信号和更防御式结果解析的合适基线。
