# Prompt Optimizer v2.5.5

## 概括
- 收藏夹开始支持媒体上传、更丰富的预览卡片，以及带资源引用的 Prompt Garden 导入。
- 导入的快照资源被迁移到更持久的存储方式里，收藏预览不容易再随着会话清理而丢失。
- 浏览器侧图像服务兼容性也做了补强，同时顺手打磨了若干导入与图标细节。

## 亮点
- 收藏夹终于更像“可承载媒体内容的资产”，尤其适合保存带图片和封面的 Prompt Garden 内容。
- Prompt Garden 导入现在支持资源引用和确认式导入流程，保存体验更稳、更可控。
- 快照预览资源的持久化方式更可靠，减少了导入视觉内容后因为会话数据清理而丢失的情况。

## 产品更新
### Web
- 保存收藏时新增通用媒体上传与预览能力。
- 收藏卡片现在可以显示封面图，也为 Prompt Garden 导入补上了更丰富的预览面板。
- Prompt Garden 导入支持带收藏资源引用的保存流程，并增加了确认式导入模式，让保存动作更明确。
- 引入可插拔的收藏预览机制，让 Prompt Garden 预览能力可以持续演进，同时不把整个收藏 UI 绑死在单一集成上。

### Core/Infra
- 将快照图片资源从会话存储中隔离出来，让导入后的收藏预览更稳定可用。
- 改进了浏览器侧使用 OpenAI SDK fetch 时的 CORS 兼容性。
- 移除了 ModelScope 和 Seedream 流程中不必要的 CORS 限制标记。

## 修复
- 修复了部分图像相关请求和 SDK fetch 行为在浏览器环境下的兼容性问题。
- 修复了 `SaveFavoriteDialog` 中缺失 `NText` 导入的问题。
- 统一了圆角应用图标导出的资源表现。

## 破坏性变更 / 升级说明
- 无。现有收藏数据可以继续使用，同时会获得更好的媒体内容支持。

## 开发者说明
- 这一版是收藏夹从“偏文本记录”转向“可承载媒体与预览状态资产”的分水岭版本。
- 如果你在维护 Prompt Garden 相关集成，可以把 `v2.5.5` 视为带资源引用收藏导入的基线版本。
