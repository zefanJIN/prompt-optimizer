# Prompt Optimizer v2.9.4

## 概括
- 恢复了图生图工作区的上传图片会话持久化，刷新或恢复会话后不再轻易丢失参考图。
- 按当前文档能力校准了 Seedream 图像模型元数据，包括 Seedream 4.5 的多图支持，以及独立的 Seedream 5.0 Lite 内置配置。
- 补上了 OpenAI Responses 请求风格支持，并优化了桌面端连接本地或局域网 LLM 服务时的直连路由。
- 通过让 Run All 变体并行执行、以及让桌面端产物直接上传到 GitHub Releases，进一步降低了测试与发布链路的摩擦。

## 亮点
- 这一版主要解决“流程可靠性”问题：图生图会话恢复重新可用，Seedream 模型选项更贴近真实能力，桌面端本地部署和 Release 发布也更稳。

## 产品更新
### Desktop
- 为桌面端新增了本地/LAN LLM 端点的直连分发逻辑，在适合直连的场景下不再强制经过代理层。
- 简化了桌面端发布工作流，让构建产物可以直接上传到 GitHub Releases，而不是继续依赖之前的中转方式。

### Web
- 恢复了图生图工作区的上传图片会话持久化，已上传源图在会话恢复后可以继续使用。
- 更新了 Run All 流程，让测试区域中的多个变体可以并行执行，减少不必要的串行等待。

### Extension
- 这一版没有扩展端专属的用户可见功能更新。

### Core/Infra
- 补上了 OpenAI Responses request style，在模型配置、适配器与转换链路中都能正确识别和使用。
- 更新了 Seedream 模型元数据与默认项：Seedream 4.5 现在正确声明多图能力并使用新的尺寸 schema，Seedream 5.0 Lite 也拥有独立的内置配置和参数面。

## 修复
- 修复了图生图会话恢复时丢失已上传图片引用的问题。
- 修复了 Seedream 内置默认项与适配器元数据和当前官方能力不一致的问题。
- 修复了桌面端访问本地或局域网模型服务时仍被错误送进代理链路的问题。
- 修复了桌面端 Release 自动化中产物未直接挂载到 GitHub Release 的流程问题。

## 破坏性变更 / 升级说明
- 本次发布没有引入 API 或存储格式层面的破坏性变更。
- 如果你依赖旧的 Seedream 模型能力判断或旧尺寸预设，升级后建议重新检查已保存的图像模型配置。

## 开发者说明
- Seedream 适配器现在改为按模型 spec 驱动，而不是继续共用一套 legacy 分支，这能让后续 provider 能力更新更安全。
- OpenAI Responses 支持同时涉及适配器、配置转换、默认值与模型管理 UI；如果你维护自定义 OpenAI-compatible 分支，建议一起复查 request-style 假设。
