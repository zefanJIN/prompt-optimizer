# Prompt Optimizer v2.10.3

## 概括
- 新增 LLM 输入图片格式的尽力兼容层。
- 上传、保存、收藏、导入导出的原始图片素材仍保持原格式。
- 当模型提供商只接受常见图片格式时，改善 Web、Chrome 插件和 Electron 桌面端的请求兼容性。

## 亮点
- 图生图、多图生图和图像理解现在会在请求模型前，尽力将 WebP 等非标准输入格式规范为 PNG。

## 产品更新
### Core
- 新增共享的运行时图片输入 normalizer，覆盖图像生成和图像理解请求。
- `image/png` 与 JPEG 输入会直接作为模型输入使用，并将 `image/jpg` 规范为 `image/jpeg`。
- 当当前运行时无法转换或转换失败时，会回退为原始图片，让模型提供商继续返回真实错误。

### Desktop
- 为 Electron 主进程请求加入基于 `nativeImage` 的转换器。

## 修复
- 非标准本地图片 MIME 不再在 LLM 请求兼容层处理前被本地校验直接拒绝。

## 破坏性变更 / 升级说明
- 无。

## 开发者说明
- 本次发布范围是 `v2.10.2..50b05a08`。
- Core 单测已覆盖图生图、多图生图、普通图像理解和流式图像理解的兼容路径。
