# Chrome 插件

当前 Chrome 插件是一个**轻量入口版本**。

点击工具栏图标后，它会打开扩展内部的 Prompt Optimizer 页面；核心 UI 复用主应用组件，不是一个单独设计的“网页内容增强插件”。

## 适合谁

适合：

- 主要在浏览器里工作
- 想快速打开 Prompt Optimizer
- 不想单独安装桌面版

不太适合：

- 需要访问本地模型或严格受限的内网接口
- 期待插件直接注入网页、右键菜单或页面回填
- 希望依赖 Chrome 账号自动同步全部配置

## 当前版本提供什么

- 点击扩展图标，打开独立的扩展页
- 在浏览器里使用 Prompt Optimizer 的主要工作区
- 使用本地存储保存扩展内的数据
- 适合作为“随手打开”的轻量入口

## 当前版本不应期待什么

为了避免误导，这里明确说明当前实现**没有**承诺以下能力：

- 没有右键菜单集成
- 没有地址栏命令
- 没有“插入回网页输入框”的通用功能
- 没有额外的 Chrome 同步能力说明
- 没有通过额外权限自动获得“更强跨域能力”

## 安装方式

### 方式一：商店版本

如果当前发布渠道提供 Chrome Web Store 版本，优先使用商店安装。

优点：

- 更新更省心
- 安装流程简单

### 方式二：开发者模式加载

适合：

- 想用最新代码
- 需要本地调试

#### 本地构建

```bash
pnpm install
pnpm build:ext
```

构建产物通常在：

```text
packages/extension/dist
```

#### 在 Chrome 中加载

1. 打开 `chrome://extensions/`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择 `packages/extension/dist`

## 使用方式

1. 把插件固定到浏览器工具栏
2. 点击图标
3. 新标签页中会打开扩展页
4. 像使用主应用一样进行模型配置、优化、测试和评估

## 与 Web 版 / 桌面版的区别

| 版本 | 最适合什么 | 主要限制 |
| --- | --- | --- |
| Chrome 插件 | 浏览器内快速打开 | 仍受浏览器环境限制 |
| Web 版 | 在线访问、跨设备临时使用 | 受 CORS / Mixed Content 影响更明显 |
| 桌面版 | 本地模型、内网接口、长期使用 | 需要安装 |

## 兼容性

当前扩展清单要求：

- Chromium 内核浏览器
- Chrome 88 或更高版本

## 出问题时先看哪里

- 打不开扩展页：在 `chrome://extensions/` 中重新加载
- 功能异常：先确认模型配置和网络是否正常
- 需要连接本地模型：优先改用 [桌面版](desktop.md)
