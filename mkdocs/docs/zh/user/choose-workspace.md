# 选择工作区

如果你第一次打开 Prompt Optimizer，先别急着看所有功能，先选对工作区。

最简单的判断方法只有一句：

**看右侧到底需要什么输入。**

## 第一次使用，先这样判断

| 你的目标 | 推荐工作区 | 左侧在改什么 | 右侧主要输入 |
| --- | --- | --- | --- |
| 给模型一个稳定角色、规则和输出边界 | [系统提示词工作区](../basic/system-optimization.md) | system prompt | 一段测试文本 |
| 打磨一条直接发给模型的任务指令 | [用户提示词工作区](../basic/user-optimization.md) | user prompt | 通常不需要额外输入 |
| 把一条提示词写成可复用模板 | [变量工作区](../advanced/variables.md) | 带 `{{变量}}` 的模板提示词 | 一组变量值 |
| 在完整会话里优化其中一条消息 | [多消息工作区](../advanced/context.md) | 整段会话里的某条 `system/user` 消息 | 整段会话 + 共享变量 + 可选工具 |
| 写文生图提示词并比较出图 | [文生图工作区](../image/text2image-workspace.md) | 图像提示词 | 图像模型 |
| 基于输入图继续改图 | [图生图工作区](../image/image2image-workspace.md) | 图生图提示词 | 输入图片 + 图像模型 |
| 让多张输入图一起参与生成 | [多图生图工作区](../image/multiimage-workspace.md) | 多图关系提示词 | 至少两张输入图 + 图像模型 |

## 如果你只想快点选对，看这 7 条就够了

1. 右侧要填测试文本：通常选 **系统提示词工作区**
2. 右侧没有额外输入：通常选 **用户提示词工作区**
3. 右侧要填变量值：选 **变量工作区**
4. 右侧要跑整段会话：通常选 **多消息工作区**
5. 右侧要直接选图像模型生成图片：通常选 **文生图工作区**
6. 右侧要先上传 1 张图：通常选 **图生图工作区**
7. 右侧要先上传多张图并保持顺序：通常选 **多图生图工作区**

## 再按右侧输入判断一次

- **右侧要输入一段测试问题**：通常说明你该用 **系统提示词工作区**。因为 system prompt 需要和一条用户消息一起执行，才能看到真实行为。
- **右侧不需要额外输入**：通常说明你该用 **用户提示词工作区**。因为被执行和被测试的，就是那条用户提示词本身。
- **右侧要填变量值**：通常说明你该用 **变量工作区**。
- **右侧要跑整段会话**：通常说明你该用 **多消息工作区**。
- **右侧要选择图像模型直接生成图片**：通常说明你该用 **文生图工作区**。
- **右侧除了模型还要先上传一张图**：通常说明你该用 **图生图工作区**。
- **右侧除了模型还要先上传至少两张图，并且顺序也重要**：通常说明你该用 **多图生图工作区**。

## 如果还是不确定，再按任务本质判断

如果你写的是：

- 你是一个客服助手
- 你必须先给结论再给步骤
- 不要编造事实

优先用 **系统提示词工作区**。

如果你写的是：

- 帮我写一封邮件
- 总结这段文字
- 写一首关于秋天的诗

优先用 **用户提示词工作区**。

如果你经常会写这种模板：

```text
请你写一首关于 {{主题}} 的诗，风格是 {{风格}}。
```

优先用 **变量工作区**。

如果你的场景里同时存在：

- `system`
- `user`
- `assistant`
- tools / function calling

而且你要优化的是其中某一条消息，就不要再勉强用基础模式，直接去 **多消息工作区**。

如果你的主要目标是产出图片，而不是文字答案：

- 只有提示词，没有参考图：去 **文生图工作区**
- 已经有一张输入图：去 **图生图工作区**
- 需要让多张图一起参与生成：去 **多图生图工作区**

## 最容易混淆的 3 组区别

区别不在于“哪个更高级”，而在于你在优化什么：

- system：长期角色、规则、边界
- user：单次任务指令、表达方式、输出要求

- 用户提示词工作区：一条固定任务先写清楚
- 变量工作区：已经明确要把它做成模板，并反复替换参数

- 变量工作区：解决“一条提示词如何参数化”
- 多消息工作区：解决“整段会话里某一条消息如何在上下文中优化”

## 第一次使用，推荐按这个顺序试

1. 单条任务先从 [用户提示词工作区](../basic/user-optimization.md) 开始
2. 角色与边界类任务用 [系统提示词工作区](../basic/system-optimization.md)
3. 需要模板化复用时切到 [变量工作区](../advanced/variables.md)
4. 需要完整上下文时切到 [多消息工作区](../advanced/context.md)
5. 需要生成图片时切到 [文生图工作区](../image/text2image-workspace.md)
6. 需要基于输入图继续生成时切到 [图生图工作区](../image/image2image-workspace.md)
7. 需要多张输入图共同约束结果时切到 [多图生图工作区](../image/multiimage-workspace.md)
