# 技术支持

这页不再重复排障内容，只回答两件事：

1. 你应该先去看哪份排查文档
2. 如果准备反馈问题，应该带哪些信息

## 先做最小分流

提交问题前，建议先按这个顺序过一遍：

1. 产品理解类问题：看 [常见问题](common-questions.md)
2. 模型连不通：看 [连接问题](connection-issues.md)
3. 页面、按钮、数据状态异常：看 [故障排除](troubleshooting.md)

很多问题在这三页里已经能直接定位。

## 当前最可靠的反馈渠道

项目仓库：

`https://github.com/linshenkx/prompt-optimizer`

如果你发现的是明确的 Bug、回归问题或文档错误，建议优先在仓库里反馈。

## 提交问题时建议带上这些信息

为了让问题更容易复现，建议至少提供：

- 使用方式：Web、桌面版还是扩展版
- 操作系统和浏览器版本
- 进入的是哪个页面，例如 `basic/user` 或 `pro/variable`
- 复现步骤
- 期望结果和实际结果
- 控制台报错、请求报错或截图
- 如果和模型调用有关，补充 provider、模型名和 Base URL

## 哪些问题适合先本地排查

下面这些情况，通常先看本地配置会更快：

- API Key 没填或填错
- 模型 Base URL 配置错误
- 浏览器本地存储损坏
- 导入了错误格式的数据备份
- 桌面版本地文件权限异常

## 文档问题也欢迎直接反馈

如果你发现公开文档和当前应用行为不一致，也欢迎直接反馈。

对这个项目来说，文档准确性和功能本身同样重要。

## 当前没有在这里承诺的内容

为了避免误导，这里不再承诺以下内容：

- 专门的官方客服邮箱
- 社区群或即时聊天支持
- 企业 SLA
- 公开 API 服务
- 已排期但尚未实现的未来能力

如果这些能力后续真的对外开放，再单独补文档会更合适。
