# 故障排除

这页处理的不是“模型连不连得上”，而是更通用的异常：

- 页面空白
- 按钮没反应
- 工作区状态不对
- 历史或设置像丢了一样
- 导入导出不符合预期

如果你的问题本质上是“模型服务连不通”，请回到 [连接问题](connection-issues.md)。

## 先做 4 个基础动作

1. 刷新页面或重启应用
2. 打开 **模型管理** 重新测试连接
3. 确认当前工作区是不是你想要的那个模式
4. 确认本地数据没有被浏览器清理

## 按问题类型排查

### 页面空白、按钮没反应

优先检查：

- 浏览器版本是否过旧
- 是否有脚本拦截类扩展影响页面
- 是否刚清理过站点数据

Web 版可以先尝试：

- 强制刷新
- 无痕模式打开
- 更换 Chromium 浏览器复现

### 左侧“优化 / 分析”按钮是灰的

通常是因为左侧工作区没有有效内容。

不同模式下的目标不同：

- 基础模式：需要至少有一条提示词
- 多消息模式：需要先选中一条 `system` 或 `user` 消息
- 图像模式：需要至少有一条提示词；图生图还需要输入图

如果你不确定是不是进错了工作区，先看 [选择工作区](../user/choose-workspace.md)。

### 右侧“测试全部”或“评估”按钮不可用

最常见的原因是输入条件不完整：

- **基础 / 系统**：没有填测试文本
- **基础 / 用户**：还没有可测试的工作区提示词
- **变量模式**：缺少必要变量值
- **多消息模式**：目标消息、共享变量或上下文条件不完整
- **对比评估**：还没有足够的测试结果

如果你不确定这是“测试条件不完整”还是“语义理解错了”，先看 [测试与评估](../user/testing-evaluation.md)。

### 历史记录或设置看起来丢了

先判断你是不是：

- 清理了浏览器缓存 / 站点数据
- 切换了浏览器或浏览器配置文件
- 换了另一台设备

如果你之前做过导出，优先去 [数据管理](../basic/data.md) 里导入备份。

### 导入失败或内容不完整

检查导入文件是否来自当前产品支持的导出格式：

- 全局数据：走 [数据管理](../basic/data.md)
- 单个上下文：走上下文编辑器的标准导出格式

如果上下文里包含预定义变量名，导入时被剔除是正常行为。

## 桌面版专项排查

### 看日志

Windows 常见日志目录：

```text
%APPDATA%\PromptOptimizer\logs\
```

如果桌面版启动失败、闪退或无法更新，先看这里最有价值。

### 本地模型在桌面版里仍然连不上

如果你已经在桌面版里仍然连不上本地模型，问题通常就不再是浏览器限制，而是：

- 本地服务没启动
- 端口写错
- 模型名写错
- 防火墙或代理配置影响

这类情况继续回看 [连接问题](connection-issues.md)。

## Chrome 插件专项排查

### 插件页打不开

去 `chrome://extensions/`：

1. 找到 Prompt Optimizer
2. 点击“重新加载”
3. 再次点击工具栏图标

### 插件表现和网页文档不一致

以当前中文文档为准。插件是轻量入口，不应期待右键菜单、页面注入或自动同步等额外能力。

## 性能慢怎么办

### Web / 在线版

- 关闭多余标签页
- 清理过大的本地历史数据前先做导出
- 避免同时跑太多测试列

### 桌面版

- 先判断是模型响应慢，还是应用本身卡
- 如果只是某个模型慢，优先换模型或检查网络

## 实在还不行时

建议把下面这几项信息一起记录下来：

- 你使用的是哪个版本：Web、桌面版还是插件
- 当前工作区模式
- 使用的 provider / 模型名
- Base URL
- 报错原文
- 能否在“测试连接”阶段复现

然后去看 [技术支持](support.md)，按那里的方式整理反馈。
