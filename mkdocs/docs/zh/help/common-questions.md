## Prompt Optimizer 会把我的提示词上传到你们服务器吗？

默认不会。

当前产品是本地优先架构：

- Web / 在线版：数据保存在当前浏览器本地
- 桌面版：数据保存在当前设备本地
- 模型请求直接发往你配置的模型服务

真正接收到你提示词内容的，通常是你选择的模型服务提供商。

## Web 版是不是内置代理层？

不是。

当前公开实现里：

- Web 版只是前端页面
- 浏览器仍然会直接向模型服务发请求
- “部署了 Web 版” 不等于 “自动获得模型中转能力”

如果你总在这里混淆，建议继续看：

- [Web 版部署](../deployment/web.md)
- [Docker 基础部署](../deployment/docker-basic.md)
- [MCP 服务器](../user/mcp-server.md)

## 在线版能不能直接连本地 Ollama / LM Studio？

大多数情况下不稳定，甚至会被浏览器直接拦截。

最常见原因是：

- 在线站是 HTTPS
- 本地模型常常暴露为 `http://localhost:...`
- 浏览器会把这类请求判定为 Mixed Content

更稳妥的方案：

1. 直接使用 [桌面版](../deployment/desktop.md)
2. 或在本机 / 局域网里自部署并自行处理网络可达性

## 为什么基础 / 用户模式右侧没有测试文本输入框？

这是当前设计的一部分。

原因是：

- **基础 / 系统**：system prompt 需要配合一段用户输入才能测试
- **基础 / 用户**：被测试的就是那条用户提示词本身，所以不需要额外测试文本

如果你经常在这里卡住，直接看：

- [选择工作区](../user/choose-workspace.md)
- [测试与评估](../user/testing-evaluation.md)

## Chrome 插件是不是能右键选中文字、再一键插回网页？

当前公开版本不承诺这些能力。

插件目前更准确的定位是：

- 一个轻量入口
- 点击图标后打开独立扩展页
- 在浏览器环境里使用 Prompt Optimizer

## 如何备份数据？

优先使用应用内的 [数据管理](../basic/data.md)：

1. 导出 JSON 备份
2. 在另一台设备导入

不要把“清浏览器缓存”当成备份方案。

## 我应该选哪个运行方式？

### 选 Web / 在线版

当你：

- 主要连接公开 HTTPS API
- 想快速体验
- 不想安装软件

### 选桌面版

当你：

- 连接本地模型
- 连接内网或自定义接口
- 需要更稳定的长期使用体验

### 选 Chrome 插件

当你：

- 主要在浏览器里工作
- 只需要一个轻量入口

### 选 Docker

当你：

- 想自托管
- 想同时获得 Web 页面和 `/mcp`
