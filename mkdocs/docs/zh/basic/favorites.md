# 收藏与导入

收藏功能适合做这几件事：

- 把已经验证过的提示词单独沉淀下来
- 给提示词补标题、说明、分类和标签
- 在不同工作区之间快速复用
- 单独导入导出一组收藏，而不动整包数据

它不是历史记录的别名。

收藏也不绑定某一种提示词来源。你可以保存手写提示词、模板改写结果、本地导入内容，也可以保存来自 Prompt Garden 提示词库的提示词。

## 最简单的理解

- **历史记录**：更像版本链和工作过程
- **收藏**：更像你主动挑出来的可复用成果

## 收藏入口在哪里

收藏相关入口主要有两类：

### 顶部工具栏里的收藏管理

这里适合：

- 搜索
- 筛选
- 浏览全部收藏
- 导入导出
- 管理分类和标签

### 工作区或结果区域里的“保存收藏”

这里适合把当前这条内容直接沉淀下来。

保存时你可以补充：

- 标题
- 描述
- 分类
- 标签
- 所属模式
- 对于图片收藏，还可以附带图片并设置封面

## 收藏管理器当前能做什么

根据当前实现，可以确认收藏管理器支持：

- 关键词搜索
- 按分类筛选
- 按标签筛选
- 网格卡片浏览
- 预览收藏内容
- 复制内容
- 立即使用
- 编辑
- 删除
- 导入 JSON、资源包、分享 HTML、原始分享 PNG
- 导出 JSON、资源包、分享 HTML、原始分享 PNG
- 清空全部收藏
- 分类管理
- 标签管理

## 分享单个收藏

如果只想分享某一条收藏，不需要导出整组收藏。

在收藏详情里点击分享入口后，可以把当前收藏导出为：

- **HTML**：单文件网页，可以直接打开浏览、复制提示词内容，也能重新导入 Prompt Optimizer。
- **PNG**：精美长图，便于展示和传播；原始 PNG 文件内会带有可导入数据。

两种格式都会按导出时看到的内容生成。封面图、描述、提示词正文、标签、变量和产品水印会默认保留；示例和版本信息可以在导出前选择是否包含。分享前请先检查正文、示例、图片和说明里是否包含不希望公开的信息。

### 如何重新导入分享文件

访问 [Prompt Optimizer](https://prompt.always200.com/)，进入 **收藏夹 -> 导入**，在同一个上传入口选择文件即可。

当前支持：

- `.po-favorites.zip`
- `.json`
- 分享 `.html` / `.htm`
- 原始分享 `.png`

PNG 导入必须使用原始文件。截图、社交平台压缩、重新保存或格式转换都可能移除图片里的导入数据；如果要确保对方可以立即复制和确认内容，优先分享 HTML。

## “立即使用”会发生什么

收藏不是简单复制一段文本。

当前点击 **立即使用** 时，系统会尽量根据收藏里的模式信息做回填：

- 文本收藏会切到对应的 `basic` 或 `pro` 工作区
- 如果收藏记录了 `system / user` 语义，也会尽量切到对应子模式
- 图片收藏会切到 `文生图`、`图生图` 或 `多图生图`

然后再把收藏内容回填到对应工作区。

所以收藏更接近“带工作区语义的复用入口”，而不只是剪贴板。

## 收藏和历史记录该怎么分工

一个很实用的工作流是：

1. 先在左侧工作区持续优化
2. 用历史记录保留过程版本
3. 当某个结果已经足够稳定时，再保存到收藏
4. 后续优先从收藏里复用，历史记录只在需要追溯过程时再看

## 收藏和数据管理的区别

### 用收藏导入导出

适合：

- 只迁移一部分精选内容
- 分享给别人一组提示词
- 保持主环境数据尽量干净

### 用数据管理导入导出

适合：

- 整机备份
- 整体迁移
- 连模型、模板、历史一起搬走

## 还可以附带哪些信息

保存收藏对话框当前支持：

- 标题
- 描述
- 分类
- 标签
- 功能模式
- 文本模式下的优化模式
- 图像模式下的子模式
- 正文内容编辑
- 图片上传
- 设置封面图
- 删除已附带图片

这意味着收藏不仅能保存文本，还能保存一组和提示词相关的媒体信息。

如果这条收藏原本来自 `多图生图`，`立即使用` 也会尽量恢复对应的多图子模式和媒体上下文，避免你还要手动重新整理这组图片。

## 示例应用到工作区

收藏也可以附带可复现示例。应用示例时，Prompt Optimizer 会把提示词和该示例对应的变量值、参数、输入图片或媒体资源一起恢复到目标工作区。

当前支持的目标包括：

- 变量提示词工作区
- 多消息上下文工作区
- 文生图工作区
- 图生图工作区
- 多图生图工作区

当一条收藏已经带着验证过的测试样例时，你可以直接从同一份证据继续迭代，不必手动重新搭建输入。

## 关于 Prompt Garden 的说明

当前实现里，收藏预览支持可插拔扩展。收藏可以保存来自不同来源的提示词资产；当某条收藏来自 Prompt Garden 提示词库时，还可以保留导入码、来源链接、示例和媒体快照。

如果部署时启用了 `Prompt Garden` 相关集成，收藏预览区还可能显示额外的外部快照信息和媒体内容。这属于可选集成能力，不是使用收藏功能的前提。

详见 [Prompt Garden](prompt-garden.md)。

## v2.10.0 新特性

### 资源感知资产

收藏现在支持：

- **版本历史**：追踪提示词的演变过程，可以查看和恢复历史版本
- **可复现示例**：附带测试结果和上下文，方便后续复现和验证
- **媒体支持**：图片和封面图完整保存

### 来源绑定

从 Prompt Garden 导入的收藏会自动记录：

- 导入码
- 来源链接
- 导入时间

方便后续更新和验证来源。

### 完整备份

导出收藏资源包时会连同引用的图片和媒体资源一起打包。导入时自动恢复资源引用，不需要单独迁移图片文件。

适合：

- 迁移包含图片的收藏
- 完整备份收藏集合
- 分享带媒体的提示词

## 使用建议

- 历史记录留过程，收藏留结果
- 标题尽量写成未来还能看懂的名字，不要只写“测试1”
- 标签不要太多，能帮助筛选就够
- 图片收藏建议明确设置封面，否则后面浏览会不够直观
- 分享收藏 JSON 前，最好先检查说明和媒体内容里有没有敏感信息
