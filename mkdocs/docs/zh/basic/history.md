# 历史记录

历史记录入口位于页面顶部工具栏的 `历史记录` 按钮。

它用来回看你已经保存下来的优化链和版本，而不是展示当前页面上所有临时状态。

## 历史记录里会看到什么

每条历史链通常包含：

- 原始提示词
- 不同版本的优化结果
- 每个版本对应的时间
- 使用过的模型名称或模型键
- 某些迭代版本附带的迭代说明

历史记录会按最近修改时间排序，方便你从最新工作继续接手。

## 当前界面里能做什么

历史记录抽屉当前支持：

- 搜索历史内容
- 展开一条历史链查看各个版本
- 把某个版本重新加载回当前工作区
- 删除单条历史链
- 清空历史记录

## 什么内容不会自动出现在这里

需要特别注意：

- 仅存在于当前工作区、还没有保存的临时编辑，不一定会稳定出现在历史记录里
- 左侧 `分析` 产生的临时工作区状态，也不应被理解为已经正式入库的版本

如果你希望某次修改后续还能可靠复用，建议先在工作区里把它保存成版本，再依赖历史记录管理。

## 历史记录和收藏怎么分工

最简单的区分方式：

- **历史记录**：偏工作过程、版本链、回看与继续修改
- **收藏**：偏精选成果、跨场景复用、单独导入导出

如果你已经验证某条提示词后续会反复使用，通常更适合再保存一份到 [收藏与导入](favorites.md)。

## 一个常见用法

你可能会这样使用：

1. 先在左侧工作区做几轮优化
2. 选中某个满意的版本保存下来
3. 过几天打开 `历史记录`
4. 搜索关键词找到当时那条链
5. 展开版本列表，选择某个版本重新加载
6. 在新的场景下继续优化

## 搜索时能搜到什么

当前搜索主要会匹配：

- 原始提示词
- 各版本的优化文本
- 迭代备注

如果你经常保存版本，给迭代写清楚说明，会更利于后续回查。
