# 数据管理

数据管理入口位于页面顶部工具栏的 `数据管理` 按钮。

它主要负责本地数据备份与恢复，而不是在线同步。

## 这页最适合解决什么问题

- 我要导出一份完整备份
- 我要把备份导入到另一台设备
- 我要单独迁移上下文集合
- 我要确认桌面版本地数据存在哪里

## 当前界面里能做什么

### 整体数据导出

可以导出一份完整的本地备份文件，内容会覆盖多个模块的数据，例如：

- 历史记录
- 模型配置
- 自定义模板
- 用户设置
- 上下文数据

导出格式是 JSON，适合本地备份和迁移。

### 存储概览

新版数据管理器会把当前本地存储拆成更容易理解的几个区域来展示，并给出大致占用信息。

这部分最适合用来回答：

- 最近是不是哪一类数据涨得特别快
- 当前设备上的本地数据大概有多大
- 导出前是否应该先做一次清理或排查

它的重点不是做精确计费，而是帮助你在“数据越来越大”变成故障之前先发现异常。

### 整体数据导入

你也可以把之前导出的 JSON 备份重新导入当前环境。

导入时要格外注意：

- 它会影响当前本地已有的数据
- 如果备份里包含模型配置，可能也会包含本地保存的模型密钥或其他敏感设置
- 在导入别人提供的备份前，最好先自行检查文件内容

### 远程备份（v2.11.0）

数据管理器也支持部署感知的远程备份。它适合把本地应用数据、收藏和相关图片资源备份到你配置的远程存储 provider，再从备份列表中选择恢复。

使用时请注意：

- 远程备份是数据安全和迁移辅助能力，不是多人协作的云同步。
- 可用 provider 会随运行环境和部署配置变化；桌面端和 Web 端的连接方式不完全相同。
- 备份会尽量携带应用数据、收藏、图片缓存和收藏图片资源。
- 恢复前可以选择恢复范围，例如只恢复应用数据、收藏，或对应图片资源。
- 如果备份创建时已有缺失图片资源，恢复界面会提示；必要时可以只恢复 JSON 数据。

在共享或恢复远程备份前，请确认里面是否包含模型配置、密钥、个人提示词、图片素材或其他敏感内容。

### 上下文集合单独导入导出

除了整体备份，数据管理器还提供上下文集合的单独导入导出能力：

- 导出到文件
- 导出到剪贴板
- 从文件导入
- 从剪贴板导入

这部分更适合高级模式里做上下文配置迁移，而不是整个应用的完整搬家。

## 桌面版会额外显示什么

如果你使用的是桌面版，数据管理器里还会显示一些本地存储信息，例如：

- 用户数据目录路径
- 主数据文件大小
- 备份文件大小
- 打开存储目录
- 刷新存储信息

这些信息主要用于排查本地存储问题，Web 版一般不会显示。

## 启动期修复和数据管理是什么关系

从最近版本开始，应用在启动时会更主动地检查本地存储是否存在明显损坏或异常膨胀的问题。

这意味着：

- 数据管理仍然是你手动备份、导入、迁移的入口
- 启动期修复是为了尽量保证应用还能正常打开
- 某些明显异常、损坏或失去引用的数据，可能会被自动清理

所以更稳的习惯仍然是：在做大改动前先导出备份。

## 数据默认存在哪里

不同运行方式下，数据落地位置不同：

- Web 版保存在当前浏览器的本地存储环境中
- 桌面版保存在本机应用数据目录
- 扩展版保存在扩展自己的本地存储空间

当前公开文档不把它描述为“云同步”或“多人共享数据”功能。

## 数据管理和收藏导入导出的区别

这两个入口很容易混淆，但作用层级不同：

- **数据管理**：偏整包备份与恢复，会影响更大范围的本地数据
- **收藏与导入**：偏精选内容的单独迁移和分享，不等于整机备份

如果你只是想带走几条已经验证过的提示词，优先看 [收藏与导入](favorites.md)。

## 收藏完整备份（v2.10.0）

v2.10.0 开始，收藏导出会连同引用的图片和媒体资源一起打包。

这意味着：

- 导出的 JSON 文件包含所有资源
- 导入时自动恢复资源引用
- 不需要单独迁移图片文件

适合：

- 迁移包含图片的收藏
- 完整备份收藏集合
- 分享带媒体的提示词

### 与普通导出的区别

| 特性 | 普通导出 | 完整备份 |
|------|----------|----------|
| 提示词内容 | ✅ | ✅ |
| 变量定义 | ✅ | ✅ |
| 图片资源 | ❌ | ✅ |
| 媒体文件 | ❌ | ✅ |
| 文件大小 | 较小 | 较大 |

### 使用建议

- 只迁移文本提示词：使用普通导出
- 迁移包含图片的收藏：使用完整备份
- 分享给他人：使用完整备份（确保资源完整）

## 使用建议

- 做大改动前先导出一份完整备份
- 分享备份文件前，先确认其中是否含有模型配置或敏感信息
- 只想迁移上下文时，优先用“上下文集合导入导出”，不要每次都整包覆盖
- 导入失败时，先检查 JSON 是否完整，再检查文件是不是由当前应用导出的格式
