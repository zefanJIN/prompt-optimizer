# 模板管理

模板管理入口位于页面顶部工具栏的 `模板管理` 按钮。

它的作用不是直接生成结果，而是管理“左侧优化时要使用的模板”。

## 模板最适合解决什么问题

如果你经常做同一类优化，例如：

- 系统提示词工作区里的角色与规则优化
- 用户提示词工作区里的任务改写
- 变量工作区里的模板化优化
- 多消息工作区里的固定优化方法

模板能帮你把这套方法固定下来。

## 当前界面里能做什么

当前公开界面主要支持：

- 查看内置模板
- 切换内置模板语言
- 新建自定义模板
- 编辑和删除自定义模板
- 复制内置模板为自己的版本
- 在简单模板和高级消息模板之间切换
- 将简单模板迁移为高级消息模板

## 模板分类是怎么分的

模板管理器会按实际模式分组展示，当前包括：

- 基础模式系统工作区模板
- 基础模式用户工作区模板
- 基础模式迭代模板
- 高级模式系统消息工作区模板
- 高级模式变量提示词工作区模板
- 高级模式迭代模板
- 图像模式文生图优化模板
- 图像模式图生图优化模板
- 图像模式迭代模板

这意味着模板并不是“一套通吃”，而是和具体工作区绑定的。

## 简单模板和高级模板有什么区别

### 简单模板

适合一段完整文本直接作为模板内容，例如：

```text
请从任务目标、约束清晰度、输出结构和表达歧义四个方面分析下面的提示词，并给出改进建议。
```

### 高级消息模板

适合需要按 `system / user / assistant` 多条消息组织的模板。

如果你的模板已经不仅是一段文字，而是需要多条消息分工，高级模板会更合适。

## 模板怎么用到工作区

通常做法是：

1. 在模板管理器里准备好模板
2. 回到具体工作区
3. 在左侧输入区选择对应模板
4. 再发起优化或迭代

模板管理器本身负责“准备模板”，真正执行模板的是各个工作区。

## 什么时候适合自定义模板

- 团队内长期复用同一种提示词优化方法
- 你已经形成了稳定的优化 checklist
- 想把某种个人风格固定下来，例如“更严格”“更偏结构化”“更偏创作性”
- 在高级模式里，需要自己定义多消息模板

## 当前不要误解的地方

当前公开界面没有提供单个模板的可视化导入导出流程说明，因此不建议把模板管理理解成“模板市场”或“社区模板中心”。

更准确地说，它目前是：

- 内置模板浏览与选择器
- 自定义模板编辑器
- 模板复用工具

## 使用建议

- 先从复制内置模板开始，再做少量修改，通常比从零写模板更稳
- 如果模板只是一段说明，优先用简单模板
- 如果模板天然就是多条消息协作，直接用高级模板
- 模板名称尽量写成“用途 + 模式”，后面回看会更清楚
