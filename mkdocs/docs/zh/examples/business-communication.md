# 商务沟通

商务沟通类任务通常也适合从 `basic/user` 开始，因为最关键的是把这些因素讲清楚：

- 受众是谁
- 沟通目标是什么
- 语气应该怎样
- 输出结构长什么样

如果你需要固定一套“专业但克制”的长期沟通风格，再考虑配合系统提示词。

## 范式一：项目进度邮件

### 原始写法

```text
给老板写个项目进度邮件
```

### 更稳的写法

```text
请写一封项目进度汇报邮件，收件人是部门总监。

背景：
- 项目：CRM 升级
- 当前进度：65%
- 主要问题：测试延期 1 周

输出要求：
1. 先写整体进度结论。
2. 再写本周完成事项。
3. 再写风险与需要支持的事项。
4. 语气专业、简洁，不要夸大成绩。
5. 控制在 300 字以内。
```

## 范式二：客户回复

```text
请写一封回复客户投诉的邮件。

要求：
1. 先明确致歉并承认问题。
2. 用用户能理解的语言说明当前处理进展。
3. 给出明确下一步时间点。
4. 不要推责，不要使用内部术语。
5. 语气真诚、专业、克制。
```

这类任务特别适合右侧做结果评估，因为你通常很容易判断输出是否：

- 真正解决了问题
- 语气是否合适
- 有没有空话
- 有没有承诺超出事实的信息

## 范式三：会议纪要或行动项整理

```text
请根据以下会议记录整理会议纪要。

输出结构固定为：
1. 会议主题
2. 核心结论
3. 待办事项
4. 负责人
5. 截止时间

要求：
1. 不要漏掉明确行动项。
2. 不要补充记录中没有出现的决策。
3. 如果负责人不明确，直接标注“待确认”。
```

## 什么时候适合系统提示词

如果你希望模型长期保持某种商务表达规范，可以把这部分写到系统提示词里，例如：

```text
你是一名企业内部沟通助手。
所有回复都应专业、礼貌、简洁。
避免夸张营销语气，避免没有依据的承诺。
```

然后把每次具体任务写到用户提示词里。

## 商务沟通怎么评估

推荐重点看：

- 是否准确理解收件人与沟通目标
- 是否突出重点，避免废话
- 是否语气合适
- 是否遵守格式要求
- 是否出现不应有的承诺、解释或题外延伸
