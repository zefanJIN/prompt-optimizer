# 创意写作

创意写作类任务通常最适合从 `basic/user` 起步，因为核心问题往往是：

- 题材怎么说清楚
- 风格怎么约束
- 结构和长度怎么写明确

如果后面发现同一套写法要反复套在不同主题上，再考虑切到变量工作区。

## 范式一：短诗或短文

### 原始写法

```text
写一首诗
```

### 更稳的写法

```text
请写一首关于春夜小雨的现代短诗。
要求：
1. 全诗控制在12行以内。
2. 语言清新克制，不要堆砌华丽辞藻。
3. 通过具体意象呈现情绪，不要直接解释主题。
4. 只输出诗作本身。
```

### 适合模式

- 初次优化：`basic/user`
- 结果比较：右侧多列测试

## 范式二：可复用的小说开头模板

如果你经常换主题、主角或叙事风格，可以把提示词改成变量化：

```text
请以{{叙事视角}}写一个小说开头。
主题是{{主题}}，整体风格偏{{风格}}。

要求：
1. 第一段就出现冲突或悬念。
2. 不要先讲大道理，先给场景。
3. 控制在{{长度}}字以内。
4. 结尾留一个继续读下去的钩子。
```

### 适合模式

- 结构固定、内容常变：`pro/variable`

## 范式三：角色口吻稳定的创作

如果你希望模型始终以某种作者身份创作，例如“古典诗人”“儿童绘本作者”“品牌文案主笔”，可以把角色约束放到系统层。

系统提示词示例：

```text
你是一名擅长克制、留白和意象表达的中文写作者。
写作时避免直接解释主题，不要使用口号式表达。
```

用户提示词示例：

```text
请写一段关于深夜办公室的短篇开头，突出疲惫和荒诞感。
```

### 适合模式

- 角色稳定 + 任务变化：`basic/system` 或 `pro/multi`

## 创意写作怎么评估

右侧比较结果时，建议优先看：

- 是否准确执行了题材和体裁要求
- 是否守住了长度和格式限制
- 是否真的体现了指定风格
- 是否有不该出现的解释、尾注或题外话

如果你关心某个点，比如“不要解释”或“开头必须更有画面感”，评估时可以加聚焦说明。
